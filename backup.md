## 	c074. 00441 - Lotto 
>題目連結:https://zerojudge.tw/ShowProblem?problemid=c074  
>☆ ☆ ☆ ★ ★
>
大略解法介紹:使用遞迴進行枚舉，以下給大概遞迴實作作法
```
int input[100];
void dfs(int data[],int index,int now,int limit)
//data[]是放你所選的數字，這邊如果你只是要求加總相乘可用int取代
//index是代表你目前放了多少變數，初始遞迴值同常設0
//now是表示現在要決定要不要選的那項的索引值
//limit通常代表題目希望你選幾項的，或遞迴終止值
{
    if(index==limit)
    {
        //他是枚舉答案之一; 
        return; //因為已經選的項已經夠了，停止這個遞迴
    }
    if(now>100)
    {
         return;//沒有下一個可以選擇的項了停止選擇    
    }
    
    bfs(data,index,now+1);//代表你不想選這項，所以你帶著之前已經選擇的項，去下一項做選擇
    data[index]=input[now];//選擇這項
    bfs(data,index,now+1);//代表你想選這項，所以你帶著之前已經選擇的項，去下一項做選擇
}
```

## 	c471. apcs 物品堆疊 (Stacking)
>題目連結:https://zerojudge.tw/ShowProblem?problemid=c471  
>題目難度:☆ ☆ ☆ ★ ★
>
大略解題方向:stack  

stack<type_name> name;
這東西就像蓋東西一樣，從底層開始建築，用於特殊題目，
基礎宣告需要一個東西type_name，簡單來說就是int long double 或自訂型態
他保證每次最上面那個都是最後放入的
以下為可以執行代碼
```
//以下為可以執行代碼
stack<int> test;//宣告他
  for(int i=0;i<5;i++)//建築我們的房子
      test.push(i);
  
  while(!test.empty())//是否為空，若不為空就執行
  {
      cout<<test.top()<<endl;//輸出最後放入的
      test.pop();//移除最後放入的
  }，
  
```
輸出結果
```
4
3
2
1
0
```
你知道嗎，其實這個題目根本不需要用到這個資料型態，這題無法用此資料型態作出答案，所以以下為真的提示  
你要做個自定義排序  
如果要最佳化，你倆倆比較哪塊放放上面比較好，如果比較好就交換，如果沒有比較好的話就不要交換  
達成:  
李國業:No   
張譽譯:No   
劉韶軒:Yes

## tnfsh 15 打獵分配問題
>題目連結:https://toj.tfcis.org/oj/pro/15/  
>題目難度:☆ ☆ ☆ ★ ★   
>
大略解題方向:自訂sort   
map<typename1,typename2> name;  
     這東西允許你使用各種不同資料型態作為索引值
內部如何做出這個有點難，如果你沒有興趣就不要查了  
關鍵詞:hash table or hashmap  

以下為可以執行代碼
```
//以下為可以執行代碼
map<int,string> test;//宣告他
  for(int i=0;i<5;i++)//設定值
        test[i]=to_string(i*i*i);
  for(auto i:test)//輸出現在全部資料
      cout<<i.first<<" "<<i.second<<endl;
  cout<<test.count(3)<<endl;//若回傳1代表這個索引值存在
  test.erase(3);//若刪除3這個索引值的資料
  for(auto i:test)//輸出現在全部資料
        cout<<i.first<<" "<<i.second<<endl;
```
輸出結果
```
0 0
1 1
2 8
3 27
4 64
1
0 0
1 1
2 8
4 64
```
